# Copy Editing

## Purpose

Take existing marketing copy and make it sharper, clearer, and more persuasive. This skill is about refinement — not rewriting from scratch. The goal is to preserve the author's intent and voice while removing everything that weakens the message.

## Step 1: Understand Before Editing

Before making a single change, answer these questions:

1. **What is this copy for?** (landing page, email, ad, product page, blog)
2. **Who is the reader?** (role, awareness level, relationship to brand)
3. **What action should the reader take?** (sign up, click, reply, buy)
4. **What voice and tone is the brand going for?** (formal, casual, technical, friendly)

Ask the user for this context if it is not provided. Editing without understanding the audience is guessing.

## Step 2: Line-by-Line Editing Checklist

Run every sentence through these filters, in this order:

### Filter 1: Does This Sentence Earn Its Place?
- Does it advance the argument, provide proof, or move the reader toward the CTA?
- If you remove this sentence, does the piece lose anything?
- If the answer is no, cut it.

### Filter 2: Is It Clear on First Read?
- Can a reader understand this sentence without re-reading?
- Is the subject-verb-object structure obvious?
- Are there any ambiguous pronouns? ("It," "this," "that" — what does each refer to?)

### Filter 3: Is It Specific?
- Replace vague claims with concrete ones.
- Before: "We help companies grow faster."
- After: "We help B2B SaaS companies increase demo bookings by 40%."

### Filter 4: Is It Concise?
- Can you say the same thing in fewer words without losing meaning?
- Before: "We are able to provide you with the ability to track your metrics."
- After: "Track your metrics."

### Filter 5: Is It Active?
- Is the subject doing the action, or is the action being done to something?
- Before: "Reports are generated by the platform automatically."
- After: "The platform generates reports automatically."

### Filter 6: Does It Sound Human?
- Read it out loud. Would a real person say this to a colleague?
- If it sounds like a press release, rewrite it.

## Step 3: Readability Improvements

### Sentence Length
- Flag any sentence over 25 words. Most can be split or trimmed.
- Vary sentence lengths. Three long sentences in a row will lose the reader.
- After a complex idea, follow with a short sentence. It gives the reader a foothold.

### Paragraph Length
- No paragraph should exceed 3 sentences in marketing copy.
- Single-sentence paragraphs are effective for emphasis.
- Use line breaks liberally — white space is a readability tool.

### Structure and Scannability
- Ensure subheadings are descriptive, not decorative.
  - Bad: "Our Approach"
  - Good: "Ship campaigns 3x faster with automated workflows"
- Convert any list of 3+ items from inline text to bullet points.
- Bold key phrases that a scanner should catch. A reader who only reads bold text should get the core message.

### Reading Level
- Aim for 6th-8th grade reading level for most marketing copy.
- This does not mean dumbing it down — it means using clear, direct language.
- Replace multi-syllable words with simpler alternatives when they carry the same meaning.

## Step 4: Tone Consistency

### Identify Tone Shifts
- Read the full piece and note where the tone changes unexpectedly.
- Common problems:
  - Casual intro that shifts to corporate-speak in the middle.
  - Confident product claims followed by hedging language ("we think," "we believe," "perhaps").
  - Technical jargon in a piece aimed at non-technical readers.

### Tone Alignment Rules
- **Confident, not arrogant:** State what the product does. Don't claim to be the best unless you prove it.
- **Conversational, not sloppy:** Write like you speak, but edit out the filler.
- **Professional, not stiff:** Avoid passive voice, nominalizations, and corporate speak.
- **Empathetic, not patronizing:** Acknowledge the reader's problem without being condescending.

### Consistency Checks
- Are contractions used consistently? (Don't mix "you will" and "you'll" without reason.)
- Is the perspective consistent? (Don't switch between "you," "we," and "one.")
- Is the formality level consistent throughout?

## Step 5: Jargon Removal

### When to Cut Jargon
- If the audience would not use this word in conversation, replace it.
- If the word is used to sound impressive rather than communicate, replace it.
- If a simpler word exists with the same meaning, use it.

### Common Jargon Replacements
| Jargon | Plain Language |
|--------|---------------|
| Leverage | Use |
| Utilize | Use |
| Facilitate | Help, enable |
| Optimize | Improve |
| Streamline | Simplify, speed up |
| Synergy | Working together |
| Ecosystem | Platform, tools, network |
| Paradigm | Model, approach |
| Ideate | Brainstorm, come up with ideas |
| Deliverables | Work, output, results |
| Bandwidth | Time, capacity |
| Circle back | Follow up |
| Move the needle | Make a difference, improve results |
| Best-in-class | [Remove — prove it instead] |
| End-to-end | Complete, full |
| Robust | Strong, thorough, reliable |
| Scalable | [Describe the specific scaling capability] |
| Seamless | Smooth, easy |
| Cutting-edge | [Remove — describe what's actually new] |

### When Jargon Is Acceptable
- The audience genuinely uses and expects the term (e.g., "API" for developers, "ROAS" for marketers).
- The term is more precise than any plain-language alternative.
- You are writing for a deeply technical audience and simplifying would reduce accuracy.

## Step 6: Passive Voice Detection

### How to Spot It
Passive voice follows this pattern: [subject] + [form of "to be"] + [past participle]
- "Reports are generated by the system." (passive)
- "The system generates reports." (active)

### When Passive Voice Is Wrong
- When it obscures who is doing the action.
- When it makes sentences longer without adding meaning.
- When it weakens a claim: "Results were achieved" vs. "Customers achieved results."

### When Passive Voice Is Acceptable
- When the actor is unknown or irrelevant: "The data is encrypted at rest."
- When the object is more important than the actor: "Your account has been upgraded."
- When active voice would sound awkward or accusatory: "Mistakes were made" (in certain diplomatic contexts).

## Step 7: Redundancy Elimination

### Common Redundancies to Cut
| Redundant Phrase | Replace With |
|-----------------|-------------|
| Absolutely essential | Essential |
| Added bonus | Bonus |
| Advance planning | Planning |
| Basic fundamentals | Fundamentals |
| Brief summary | Summary |
| Collaborate together | Collaborate |
| Completely eliminate | Eliminate |
| End result | Result |
| Final outcome | Outcome |
| Free gift | Gift |
| Future plans | Plans |
| In order to | To |
| New innovation | Innovation |
| Past history | History |
| Plan ahead | Plan |
| Repeat again | Repeat |
| Still remains | Remains |
| Unexpected surprise | Surprise |
| Very unique | Unique |

### Structural Redundancy
- Saying the same thing twice in different words within the same section.
- Repeating the value proposition verbatim instead of reinforcing it from a new angle.
- Multiple sections covering the same point — consolidate or cut.

### Qualifier Redundancy
- Remove words that add no meaning: "really," "very," "just," "quite," "rather," "somewhat," "actually," "basically," "literally" (when not literal).
- These words weaken rather than strengthen: "We are very committed" is weaker than "We are committed."

## Step 8: Power Word Substitution

### Weak Verbs to Strengthen
| Weak | Strong Alternatives |
|------|-------------------|
| Help | Accelerate, unlock, enable, equip |
| Make | Build, create, craft, design |
| Get | Earn, gain, capture, secure |
| Do | Execute, deliver, achieve, ship |
| Use | Deploy, apply, harness |
| Give | Deliver, provide, grant |
| Have | Own, control, command |
| Try | Test, explore, experiment with |
| Know | Understand, master, uncover |
| Think | Believe, expect, recognize |

### Weak Adjectives to Replace
| Weak | Stronger Options |
|------|-----------------|
| Good | Proven, reliable, effective |
| Great | Outstanding, exceptional (with proof) |
| Nice | [Cut — be specific about what's nice] |
| Big | Significant, substantial, [use a number] |
| Fast | Instant, real-time, [use a timeframe] |
| Easy | One-click, no-code, 2-minute setup |
| New | [Describe what changed and why it matters] |
| Important | Critical, essential, [show why it matters] |

### A Warning on Power Words
- Do not substitute every weak word. Overusing strong words creates breathless, hype-filled copy.
- Use strong words at key moments: headlines, CTAs, opening sentences, proof points.
- Let simple words carry the load in supporting sentences.

## Step 9: Before/After Examples

### Example 1: SaaS Homepage Headline

**Before:**
"Our innovative platform leverages cutting-edge AI technology to help businesses optimize their workflow processes and achieve better outcomes."

**After:**
"Ship work faster. AI that automates your team's repetitive tasks — so you focus on what matters."

**What changed:** Removed jargon (leverage, cutting-edge, optimize), replaced vague "better outcomes" with a specific benefit, shortened from 22 words to 18, made it active and direct.

### Example 2: Feature Description

**Before:**
"With our robust and scalable analytics solution, data-driven insights are surfaced in real-time, enabling stakeholders to make more informed decisions and drive strategic initiatives forward across the entire organization."

**After:**
"See what's working in real time. Your team gets live dashboards and alerts — no waiting for weekly reports, no digging through spreadsheets."

**What changed:** Eliminated passive voice ("insights are surfaced"), removed buzzwords (robust, scalable, stakeholders, strategic initiatives), added concrete details (dashboards, alerts, weekly reports), wrote in second person.

### Example 3: CTA Section

**Before:**
"If you would like to learn more about how our solution can help your organization achieve its goals, please do not hesitate to reach out to our team of experts who would be happy to schedule a personalized demonstration at your earliest convenience."

**After:**
"See it in action. Book a 15-minute demo — we'll show you exactly how it works for your team."

**What changed:** Cut from 42 words to 19. Replaced corporate politeness with direct action. Added specificity (15-minute). Made the benefit concrete (how it works for your team).

### Example 4: Testimonial Cleanup

**Before:**
"We've been really happy with the product. It's been great for our team and has helped us do things much better and faster than before. Would definitely recommend."

**After:**
"Our team cut project turnaround from 2 weeks to 3 days. We shipped 4 campaigns last month that would have taken a full quarter before."

**What changed:** Replaced vague praise with specific metrics. Added a concrete example. (Note: when editing testimonials, verify specific claims with the customer.)

## Step 10: Output Format

### Deliver Edits With:

1. **Edited copy** — the full revised text, clearly formatted.
2. **Change summary** — a brief list of the major changes made and why.
3. **Tracked changes** — for substantial edits, show before/after for key passages so the user can see the reasoning.
4. **Questions** — flag anything that requires the user's input (unclear claims, missing specifics, ambiguous audience).

### Formatting Convention
When showing before/after comparisons inline:
```
~~Original text that was changed~~ → Revised text with improvements

Or for longer passages:

**Before:**
[Original text]

**After:**
[Revised text]

**Why:** [1-sentence explanation]
```

## Common Copy-Editing Mistakes

1. **Over-editing voice out of the copy.** Preserve the author's personality. Edit for clarity, not conformity.
2. **Adding words instead of cutting them.** Editing should almost always make copy shorter.
3. **Fixing grammar but missing strategy.** A grammatically perfect sentence that says nothing is still bad copy.
4. **Ignoring context.** Copy that seems "too casual" might be perfect for a Slack-integrated product. Ask about the audience before making tone changes.
5. **Making every sentence punchy.** Varied rhythm is more readable than relentless short sentences.
6. **Removing repetition that serves a purpose.** Some repetition is intentional — reinforcing key messages across sections is a feature, not a bug. Cut redundancy within sections, not strategic repetition across them.
